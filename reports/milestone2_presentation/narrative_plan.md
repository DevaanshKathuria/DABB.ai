# Milestone 2 Presentation Plan

## Audience
- Course staff reviewing the final project submission
- Classmates during the demo presentation

## Objective
- Explain how the Milestone 1 baseline evolved into a public-hostable Milestone 2 release
- Show the assistant-backed reporting workflow, deployment path, and evaluation artifacts

## Narrative Arc
1. Problem and why contract risk analysis matters
2. What Milestone 2 adds beyond the baseline
3. How the system is wired end to end
4. How the hosted app is used
5. What the evaluation and artifacts show
6. Known limitations and release safeguards
7. Final submission checklist

## Slide List
1. Title slide
2. Milestone 2 scope and value
3. End-to-end architecture
4. User flow and public deployment
5. Results and confusion matrix
6. Limits, safeguards, and reproducibility
7. Final checklist / closing slide

## Visual System
- Palette: deep navy, slate, teal, and warm amber accents
- Layout: bold title zones, card-based content, and clean spacing
- Tone: technical but presentation-friendly, with no cluttered boilerplate

## Source Plan
- Use the local confusion matrix image generated by the evaluation CLI
- Use metric values from `reports/logreg_summary_metrics.csv`
- Use short deployment callouts from `docs/deployment.md`

## Editability Plan
- Keep all headings, body text, bullets, and metric cards as editable PowerPoint objects
- Use the confusion matrix as a single placed image
- Avoid rasterizing the whole slide so the deck stays easy to revise
